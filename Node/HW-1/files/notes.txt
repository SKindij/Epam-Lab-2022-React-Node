Впровадити веб-сервер, який розміщує та обслуговує файли

Вимоги:
Використовуйте стандартний http-модуль або експрес-фреймворк для реалізації простого веб-сервера;
Використовуйте модуль fs для створення/зміни/читання файлів у файловій системі;
Записуйте інформацію про кожен запит у журнали;
Програма має підтримувати розширення файлів log, txt, json, yaml, xml, js (зверніть увагу, що назва файлу може містити символ .);
Будь ласка, переконайтеся, що вашу програму можна запустити за допомогою команд npm install і npm start відразу після git pull;
Програма має працювати на порту 8080.

Критерії приймання:
Сервер зберігає файл на запит createFile і відповідає зі статусом 200, використовуйте параметри імені файлу та тіла вмісту для передачі даних файлу.
Сервер повертає список завантажених файлів на запит getFiles
Сервер повертає вміст файлу на запит getFile. Використовуйте параметр URL-адреси імені файлу, щоб визначити, який файл ви хочете отримати.
Якщо файл із вказаною назвою не знайдено, поверніть статус 400.

Додаткові критерії:
Сервер обробляє помилки та перевіряє вхідні параметри для всіх запитів.
Ви можете додати захист паролем для файлів -
з додатковим (необов’язковим) паролем параметра в запиті createFile,
і додатковий (необов’язковий) пароль параметра в запиті getFile у параметрах запиту url.
Вважайте, що сервер повинен обробляти недійсний і порожній пароль для захищених файлів.
Можливість змінювати вміст файлів (додаткова кінцева точка);
Можливість видалення файлів (додаткова кінцева точка);
